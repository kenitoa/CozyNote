# 위험 요소 및 재발 방지 메모

## UI 레이아웃 위험

- 오른쪽 패널 폭을 줄이면 미니게임처럼 내부 기준 폭이 고정된 컴포넌트가 쉽게 깨진다.
- 단순히 창 크기만 키우면 세로 overflow는 해결되지 않는다. 긴 오른쪽 패널은 스크롤 구조가 필요하다.
- 앨범 이미지는 컨테이너 중앙 정렬과 이미지 비율 보존을 함께 적용해야 대칭이 맞는다.

## 스타일 우선순위 위험

- 다크 테마의 전역 `.label` 규칙이 밝은 카드 내부 글자까지 흰색으로 덮을 수 있다.
- 밝은 카드가 다크 테마 안에서도 유지되어야 하는 경우, 카드 내부 텍스트 색을 더 구체적인 selector로 다시 지정해야 한다.
- 공통 버튼 스타일을 직접 바꾸면 편집기, 설정, 사이드바까지 영향을 줄 수 있으므로 음악 플레이어 전용 클래스로 제한해야 한다.

## 실행 환경 위험

- `winget`, Maven, 외부 Maven 캐시는 사용자 환경이나 샌드박스 권한에 따라 보이지 않을 수 있다.
- `run.ps1` fallback은 JavaFX/Jackson/SQLite jar 캐시 상태에 의존한다.
- 실행 검증에서 GUI 앱은 명령 시간 제한 때문에 정상 실행 여부와 프로세스 종료 여부를 분리해서 판단해야 한다.

## 파일 구조 위험

- 루트 정리 중 `playlist/Good Days.mp3`가 다른 프로세스에 잠겨 이동/삭제되지 않았다.
- 잠긴 파일은 권한 상승으로도 해결되지 않을 수 있으며, 해당 파일을 사용하는 앱이나 Java 프로세스를 닫은 뒤 다시 정리해야 한다.
- 파일 이동 후 IDE의 열린 탭 경로가 예전 `src/...`, `build.gradle`, `run.ps1`를 가리킬 수 있으므로 실제 경로가 `app/...`인지 확인해야 한다.

## 인코딩 위험

- PowerShell에서 Markdown, FXML, Java 파일을 읽을 때 기본 인코딩으로 보면 한글이 깨져 보일 수 있다.
- FXML 검증은 `-Encoding UTF8`을 명시해야 잘못된 파싱 오류를 피할 수 있다.
