# 2026-06-24 작업 로그

## 실행 구조 정리

- 루트 실행 진입점으로 `CozyNote.cmd`를 추가했다.
- `CozyNote.cmd`에서 JDK 설치 여부를 확인하고, 없으면 `winget`으로 Temurin JDK 21 설치를 시도하도록 했다.
- Maven이 있으면 `mvn javafx:run`으로 실행하고, Maven이 없으면 `app/run.ps1`로 fallback 실행하도록 구성했다.
- 프로젝트 소스, 문서, 리소스, 빌드 관련 파일을 `app` 폴더 아래로 이동했다.
- 루트에는 실행용 CMD만 남기는 방향으로 정리했으나, `playlist/Good Days.mp3`는 다른 프로세스가 사용 중이라 루트 원본 제거가 완료되지 않았다.

## 초기 화면 크기 및 레이아웃

- 앱 시작 시 사용 가능한 화면 크기를 기준으로 창 크기를 잡도록 변경했다.
- 화면이 기본 시작 크기보다 작으면 창을 최대화하도록 했다.
- 초기 폭이 부족할 때 왼쪽 사이드바를 자동으로 접어 편집기와 오른쪽 패널이 잘리지 않도록 했다.
- 오른쪽 패널 폭 계산을 실제 UI 폭과 맞춰 레이아웃 폭 계산 오류를 줄였다.

## 오른쪽 음악 플레이어 UI

- 음악 플레이어 영역을 단순 카드에서 MP3 플레이어 형태로 재구성했다.
- 구조를 `상단 바 -> 앨범 화면 -> 웨이브폼 -> 하단 컨트롤 덱` 형태로 바꿨다.
- 색상은 CozyNote의 베이지, 우드, 포인트 블루 계열로 통일했다.
- 앨범 이미지를 중앙 배치하고 비율 보존 방식으로 표시하도록 했다.
- 앨범 이미지 프레임을 아이보리/우드 톤으로 조정했다.
- 재생, 이전/다음, 반복, 메뉴 버튼과 슬라이더를 음악 플레이어 전용 스타일로 분리했다.
- 오른쪽 패널 내용이 세로로 넘칠 때 겹치지 않도록 스크롤 가능한 구조로 바꿨다.

## 텍스트 색상 보정

- 다크 테마 전역 `.label` 규칙 때문에 오른쪽 패널 텍스트가 흰색으로 덮이는 문제가 있었다.
- 오른쪽 `widget-panel` 내부 기본 글자색을 브라운 계열로 고정했다.
- MP3 상단 파란 헤더 안의 텍스트만 흰색으로 유지하도록 예외 처리했다.
- 미니게임 패널 제목, 요약, 레벨 텍스트도 밝은 카드 배경에 맞는 브라운 계열로 재지정했다.

## 검증

- FXML 5개 파일 파싱을 확인했다.
- Java 소스 컴파일을 확인했다.
- GUI 화면의 최종 시각 검증은 사용자가 제공한 스크린샷을 기준으로 반복 조정했다.
